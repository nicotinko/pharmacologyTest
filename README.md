# Pharmacology Test Generator

Этот проект предоставляет автономную систему для генерации интерактивных HTML-тестов по фармакологии из Markdown-файлов. Тесты полностью локальны, не требуют бэкенда или внешних зависимостей, что делает их идеальными для публикации на GitHub Pages.

## Структура Проекта

```
.
├── source/                 # Исходные Markdown/TXT файлы с вопросами
├── json/                   # Сгенерированные JSON-файлы с банками вопросов
├── tests/                  # Сгенерированные HTML-файлы тестов
├── .gitignore              # Исключения для Git
├── catalog.html            # Каталог всех сгенерированных тестов
├── index.html              # Главная страница со списком тестов
├── parser.py               # Основной Python-скрипт для парсинга и генерации
└── README.md               # Этот файл
```

*   **`source/`**: В этой папке хранятся ваши исходные файлы вопросов в формате Markdown (`.md`) или TXT (`.txt`). Каждый файл представляет собой отдельную тему теста.
*   **`json/`**: Здесь будут храниться промежуточные JSON-файлы, сгенерированные скриптом `parser.py`. Каждый JSON-файл содержит структурированные данные для одного теста.
*   **`tests/`**: В этой папке находятся финальные интерактивные HTML-файлы тестов, готовые к использованию.
*   **`parser.py`**: Главный скрипт проекта, который парсит исходные Markdown-файлы, создает JSON-банки вопросов и генерирует HTML-тесты, а также `catalog.html` и `index.html`.
*   **`index.html`**: Главная страница проекта со списком всех доступных тестов.
*   **`catalog.html`**: Альтернативная страница-каталог со списком всех тестов в табличном формате.

## Подготовка Базы Вопросов (Markdown/TXT)

Для создания тестов используйте Markdown (`.md`) или TXT (`.txt`) файлы в папке `source/`. Каждый файл должен содержать вопросы по одной теме.

**Формат вопросов:**

*   Каждый вопрос начинается с заголовка Markdown второго уровня (`## N. `), где `N` — порядковый номер вопроса (например, `## 1.`).
*   Варианты ответов перечисляются с префиксом буквы и скобки (например, `a)`, `б)`).
*   **Правильные варианты ответов** помечаются двумя звездочками в конце (`**`).
*   **Пояснения** к вопросу должны быть заключены в двойные квадратные скобки `[[Ваше пояснение]]` и располагаться сразу под вопросом (после вариантов ответа, если это множественный выбор, или сразу после строки вопроса, если это заполнение пропусков). Если пояснение отсутствует, соответствующий блок в HTML не генерируется.

**Примеры:**

**Множественный выбор:**
```markdown
## 1. Какие из перечисленных препаратов относятся к бета-адреноблокаторам?
a) Атенолол**
b) Сальбутамол
c) Пропранолол**
d) Изопреналин
[[Атенолол и Пропранолол являются бета-адреноблокаторами. Сальбутамол — бета-2-адреномиметик, Изопреналин — неселективный бета-адреномиметик.]]
```

**Заполнение пропусков:**
```markdown
## 2. Фермент {моноаминоксидаза|МАО} участвует в метаболизме катехоламинов.
[[Моноаминоксидаза (МАО) — это фермент, играющий ключевую роль в деградации моноаминов, включая катехоламины и серотонин.]]
```
Для вопросов на заполнение пропусков, правильные варианты ответа (включая синонимы) заключаются в фигурные скобки `{...}` и разделяются символом `|`.

## Генерация Тестов

Для генерации HTML-тестов, JSON-банков вопросов, а также `index.html` и `catalog.html`, выполните Python-скрипт `parser.py`:

```bash
python parser.py
```

Скрипт автоматически:
1.  Парсит все `.md` или `.txt` файлы из папки `source/`.
2.  Извлекает вопросы, варианты ответов, правильные ответы и пояснения.
3.  Определяет тип вопроса (одиночный/множественный выбор, заполнение пропусков).
4.  Генерирует JSON-файлы с банками вопросов в папке `json/`. Имена файлов будут транслитерированы и приведены к нижнему регистру (например, `adrenolitiki.json`).
5.  Генерирует интерактивные статические HTML-файлы тестов в папке `tests/`. Имена файлов также транслитерированы и приведены к нижнему регистру (например, `adrenolitiki.html`).
6.  Создает `index.html` и `catalog.html` с актуальными ссылками на все сгенерированные тесты.

## Логика Кода и Структура

### `parser.py`

*   **`transliterate(name)`**: Функция для транслитерации русских названий в латинские. Обеспечивает, что имена файлов будут в нижнем регистре.
*   **`parse_md_file(file_path)`**: Основная функция для чтения Markdown/TXT файла, извлечения вопросов, вариантов ответов, правильных ответов, пояснений и определения типа вопроса. Возвращает структурированный словарь данных теста.
*   **`generate_html_test(test_data, output_path, prev_test_link, next_test_link, main_page_link)`**: Генерирует одиночный интерактивный HTML-файл теста. Встраивает CSS и JavaScript.
    *   **Отображение вопросов:** Использует `radio` для одиночного выбора, `checkbox` для множественного выбора и текстовые поля для заполнения пропусков.
    *   **Визуальный фидбек:** При нажатии "Проверить" правильные ответы подсвечиваются зеленым, неправильные — красным. Пропущенные правильные ответы обводятся зеленой рамкой. На карточке вопроса появляется зеленая галочка (&#10004;) при правильном ответе на весь вопрос или красный крестик (&#10006;) при ошибке.
    *   **Пояснения:** Отображаются после нажатия "Проверить" (если предоставлены в `.md` файле) с жирным заголовком "Объяснение:".
    *   **Навигация:** Под кнопками "Проверить" и "Сбросить" расположен блок навигации с ссылками "Предыдущий тест", "Все тесты" и "Следующий тест". Ссылки на нерабочие (первый/последний) тесты не генерируются.
*   **`process_all_source_files(source_dir, json_dir, tests_dir)`**: Оркестратор, который проходит по всем исходным файлам, парсит их, генерирует JSON и HTML, а также собирает метаданные для `catalog.html` и `index.html`. Также определяет ссылки для навигации между тестами.
*   **`generate_catalog_html(test_metadata, output_path)`**: Генерирует `catalog.html` со списком всех тестов в табличном формате.
*   **`generate_index_html(test_metadata, output_path)`**: Генерирует `index.html` с заголовком "Тесты по фармакологии для самоконтроля знаний" и списком ссылок на все тесты.

### CSS (`style` блок в HTML)

*   Определяет внешний вид страниц тестов, включая контейнеры, заголовки, блоки вопросов, варианты ответов, пояснения, кнопки и результаты.
*   Содержит стили для подсветки правильных/неправильных ответов и обводки пропущенных правильных ответов.
*   Определяет стили для иконок статуса вопроса и навигационного блока.

### JavaScript (`script` блок в HTML)

*   **`quizData`**: Встроенный JSON-объект, содержащий все данные для текущего теста.
*   **`renderQuiz()`**: Функция, которая динамически строит HTML-форму теста на основе `quizData` при загрузке страницы.
*   **`checkAnswers()`**: Функция, которая обрабатывает ответы пользователя, подсчитывает результаты, применяет визуальный фидбек (подсветка, обводка, иконки статуса) и отображает пояснения.
*   **`resetQuiz()`**: Функция для сброса формы, удаления визуального фидбека и скрытия пояснений. Также прокручивает страницу в начало.

## Использование на GitHub Pages

Проект предназначен для публикации на GitHub Pages. Просто отправьте все сгенерированные файлы в ветку `main` вашего репозитория, и они станут доступны по адресу `https://<ваш_логин_гитхаб>.github.io/<имя_репозитория>/`.
